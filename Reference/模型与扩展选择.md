# 模型与扩展选择

这页记录的是这场多事项讨论会议里，老师关于模型和扩展的实操判断。

## 模型不是一个水平

老师这场的判断很明确：

* 默认模型可以完成一些日常沟通
* 真想更顺，`Claude` 仍然是主力
* 编码型模型不一定适合做浏览器和 APP 操作
* 有些模型一听到“写代码”就会直接冲去改代码

## 模型差异最明显的地方

这场里，模型差异主要体现在：

* 能不能听懂“先开网页再操作”
* 会不会绕过 UI 直接去写代码
* 操作 APP 时能不能稳住步骤
* 联网搜索时会不会自己走偏

## 扩展安装的建议

老师反复强调：

* 不要从网上乱下扩展
* 尽量走官方、系统内置或本地加载的路径
* 需要时就开开发者模式
* 加载本地扩展
* 再配置 token

## 版本和 profile 也会影响行为

这场里还有几个很工程化的小坑：

* 插件版本不同，行为不一样
* profile / 分组会影响控制关系
* 老重启会话，模型行为可能变
* 有些系统会 fallback 到别的模型

## 这页最值得记的一句话

不是所有模型都适合同一种任务。选错模型，效果不是“差一点”，而是工作方式都会变。
