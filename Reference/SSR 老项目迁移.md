# SSR 老项目迁移

这场课里老师给了一个很典型的案例：

* 一个老的 `create-react-app` 项目
* 代码量大
* 每天都发版
* 是公司核心收入来源
* 但又必须从 `CSR` 改成 `SSR`

它最难的地方不是“怎么写 SSR”，而是：

* 不能停机
* 不能影响业务迭代
* 不能因为改造把线上搞崩

老师的原则是：

* 能不全量重构，就不要全量重构
* 尽量保留现有路由和业务组织
* 只补服务端构建和最小 SSR 入口
