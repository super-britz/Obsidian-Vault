# 技术选型方法

这页对应老师在“技术选型”课里最核心的一条方法论。

## 老师最反对什么

老师最反对的是：

* “我喜欢这个，就用这个”
* “我看别人用了，我也用”
* “它现在最火，所以就上”

## 他真正看重什么

### 1. 项目类型

* 你做的是 `SPA`、`MPA` 还是同构
* 你要不要离线
* 你要不要缓存

### 2. 团队现实

* 团队会不会用
* 维护成本高不高
* 会不会把大家带进不必要的坑

### 3. 生态和社区

* 组件齐不齐
* 文档够不够
* 社区和成熟度够不够

### 4. 部署和运行环境

* 和 `Cloudflare` / `AWS` / `serverless` 相性好不好
* 包管理和构建链是否稳定

### 5. 你能不能解释它

老师反复强调：

* 为什么选它
* 你踩过什么坑
* 团队为什么接受它
* 如果不合适怎么回退

这些问题都是面试会问的。

## 这页最重要的一句话

技术选型不是“选会不会写”，而是“选谁来承担长期维护成本和收益”。
