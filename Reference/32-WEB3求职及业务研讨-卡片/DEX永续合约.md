# DEX永续合约

这张卡来自 [[32-WEB3求职及业务研讨｜直播笔记整理]]。

老师讲的 [[DEX永续合约]]，重点不只是页面像交易所，而是背后的交易对手方、价格源、清算和 LP 风险机制。

核心模块：

- [[LP池子]] 作为交易对手方。
- [[预言机]] 提供价格，常见组合是 [[Pyth]] / [[Chainlink]]。
- [[资金费率]] 平衡多空仓位。
- [[保证金]] 和 [[强制平仓]] 控制风险。
- [[链上清算]] 或清算机器人处理极端仓位。
- [[LP保护机制]] 防止池子被单边行情或套利打穿。

面试时要能解释：

- 用户盈利和亏损分别由谁承接。
- LP 为什么愿意存入资金。
- 极端行情下如何限制风险敞口。
- 前端如何实时展示价格、盘口、仓位和清算状态。
