# LP池子

这张卡来自 [[32-WEB3求职及业务研讨｜直播笔记整理]]。

[[LP池子]] 是 Liquidity Pool，流动性池。老师这场把它作为 DEX 永续合约的核心概念讲：用户开仓时，对手方不是另一个用户，而是池子。

基本逻辑：

- LP 存入 [[USDT]] 等资产。
- 平台铸造 LP 份额凭证，例如 [[双代币模型]] 里的 `XCLP`。
- 用户亏损时，资金进入池子。
- 用户盈利时，池子向用户支付收益。
- LP 按份额分享交易手续费、平台收益或代币激励。

风险点：

- 单边行情可能让池子持续亏损。
- 大额套利者可能利用价格差或内幕信息打穿池子。
- 提现挤兑会影响平台正常运营。

所以 LP 池通常需要配套 [[LP保护机制]]、[[预言机]]、限额、冷却期和清算规则。
