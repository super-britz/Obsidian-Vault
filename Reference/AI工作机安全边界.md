# AI工作机安全边界

这页是这场前端直播里最值得单独拎出来的风险主题。

## 核心原则

跑 agent、浏览器控制和远程环境的机器，不要和你的真实身份、真实财产、真实主力账号混在一起。

## 老师反复强调的边界

### 账号隔离

* Chrome 用新的 Google ID
* Apple 生态尽量用隔离账号
* 不要让这套环境直接接触你的主身份

### 财产隔离

* 不要登录币安或其他资金相关账户
* 不要登录能直接操作资产的网站
* 不要把财产入口交给 agent 工作机

### 插件和扩展验真

* 不要装来历不明的扩展
* 同名扩展里可能有假的
* 假扩展可能投毒、盗号、窃取数据

### 不要乱装三方 security / skill / agent 组件

老师的意思是：

* 很多“安全工具”本身也可能成为新的风险入口
* 安装前一定要看文档和权限范围

### 登录态浏览器的风险更高

默认新浏览器虽然能力弱一点，但风险也更低。

一旦你让工具接管“已登录账号”，风险等级会明显上升。

## 更稳妥的客观结论

不管你是否完全接受老师的表达，至少可以保留这几条：

* 建立专门的 AI 工作机或专门的工作环境
* 建立专门的浏览器账号
* 建立专门的测试账号
* 把个人核心资产和工作流工具隔离开
