# SSR 与 CSR

`SSR` 和 `CSR` 最常见的区别，不是“一个高级一个低级”，而是首屏内容、抓取友好度、部署复杂度和运行位置不同。

从这次课的语境看：

* `SSR` 的主要价值是两件事：
  * [[AIO 与 SEO]]
  * 性能
* `CSR` 更轻，更适合纯交互应用
* 但如果项目要靠搜索入口、国际化落地、内容抓取、AI 检索暴露，SSR 的价值会明显上升

老师在这场课里的强调点是：

* 面试里不要只回答“SSR 可以服务端渲染 HTML”
* 更要回答“它为什么对搜索、AI 爬虫、性能和业务增长重要”
