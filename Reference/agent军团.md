# agent军团

这页记录的是老师在这场会议里想搭出来的“虚拟办公室”。

## 终极目标

老师讲得很清楚：

* 你坐在电脑前，有些事你自己做更快
* 真正的价值，是你离开电脑以后，它还能继续跑

所以 agent 军团的意义不是“少点几下按钮”，而是：

* 持续工作
* 并行工作
* 可调度
* 可复用

## 现场搭的几个角色

老师现场举的例子包括：

* 会议需求开发官
* 前端情报官
* 舆论 / 币圈观察官
* 抖音二创剧本官
* 健康睡眠心情官

## 这些 agent 各自做什么

大致包括：

* 参会、整理需求、拉代码开发
* 收集前端技术发展
* 整理舆情和币圈新闻
* 分析二创话题并做视频
* 分析睡眠、心情和身体数据

## 重要的工程细节

这场里有几个很容易漏掉的点：

* agent 不是天然永久存在
* 有些状态只在内存里
* 有些会写到 workspace 或 `jobs.json`
* 如果没落盘，重启后就可能丢
* 不同模型建立 sub-agent 的方式不同
* 有的模型会只给你方案，不真正创建

## 并行和手机调度

老师强调：

* 多个 agent 可以并行
* 可以在手机上调用
* 最后要形成一个“你不在电脑前，它也能继续工作”的状态

## 风险提醒

老师也提到一个很现实的风险：

* 如果机器人入口是公开的
* 别人能 at 到它
* 就可能触达你的电脑环境

所以 agent 军团一定要做访问控制。

## 这页最重要的一句话

前面学的浏览器控制、APP 自动化、VS Code 自动化，最后都应该为 agent 军团服务。
