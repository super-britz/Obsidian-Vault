# Cloudflare Tunnel

这页对应老师在 AWS 那场课里布置的一个很实操的作业点。

## 老师这场为什么讲它

因为前面的作业已经做到：

* 域名
* EC2
* 本地服务 / 远端服务

接下来他想让大家继续补上：

* `Cloudflare`
* `Tunnel`

## 它在这场课里的典型用途

### 把本地服务带出来

例如：

* 你本地跑了一个 `127.0.0.1` 的开发服务
* 又想让外部能访问
* 就可以借 `Cloudflare Tunnel` 把它接出去

### 把 EC2 上的服务接出来

老师这场课里更偏向的作业方向是：

* 接着前面做过的 `EC2`
* 配域名
* 解析到 `Cloudflare`
* 再用 `Tunnel` 暴露服务入口

## 它和“直接暴露 IP”有什么区别

老师这里虽然没系统展开，但语境很明确：

* 不是简单裸露一个公网 IP
* 而是通过 Cloudflare 这层把入口接出来

## 这节课的作业里，它代表什么

它代表这件事：

> 不是只把服务跑起来，而是把“域名 + 入口 + 访问路径”真正接通。

## 这页最重要的一句话

`Cloudflare Tunnel` 在这场课里不是单独技巧，而是“把服务从内部环境稳定接到外部访问入口”的作业桥梁。
