# SPA 与 MPA

这页对应老师在前端架构课里对几种页面形态的基础划分。

## `SPA`

* 单页应用
* 开发最常见
* 上手简单
* 但白屏时间长是典型问题

老师特别强调：

* 做 `SPA` 不能只会脚手架
* 至少得理解构建链和白屏优化

## `MPA`

* 多页应用
* 每个路由是真路由
* 页和页之间切换时，会有更多内容重新渲染

## `MPA + SPA`

* 一部分走真实服务端路由
* 一部分走前端接管的路由
* 真实路由和假路由会混在一起

## 为什么会出现“真假路由”

老师这场课里讲得很明确：

* 前端 history 路由看起来像真 URL
* 但服务端未必真有这个路由
* 刷新就可能 `404`
* 所以部署时常常要做 fallback

## 同构 / SSR

老师这里的同构重点不是概念，而是：

* 前端开始真正和 `Node.js` 服务渲染结合
* 要同时考虑模板、路由和部署方式

## 这页最重要的一句话

`SPA` 和 `MPA` 的区别，不只是页面怎么切，而是“路由到底由谁接管”。
