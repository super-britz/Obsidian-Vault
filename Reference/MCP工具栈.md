# MCP工具栈

老师把 MCP 理解为：让 AI 以标准化方式调用外部能力，而不只是聊天。

## 直播里提到的 MCP 方向

* GitHub MCP
* Chrome / Chrome DevTools MCP
* Figma MCP

## 作用

MCP 的价值在于把这些动作标准化：

* 读项目和仓库
* 打开网页并分析
* 读取设计稿
* 调本地文件和脚本
* 把外部工具变成 AI 可调用的能力

## 直播里出现的入口

* `mcp.so`：老师把它当成 MCP 索引 / 市场入口

## 高价值场景

### GitHub MCP

* 项目创建
* 代码管理
* 让仓库动作更自然语言化

### Chrome / Chrome DevTools MCP

* 性能分析
* 火焰图解读
* Long Task 排查
* 布局和错误定位
* 真实面试题诊断

### Figma MCP

* 让设计稿直接进入代码生成链路
* 缩短设计到开发的转换距离

## 这页最重要的一句话

MCP 不是某个单独插件，而是一种“让 AI 获得外部执行能力的标准化接口”。
