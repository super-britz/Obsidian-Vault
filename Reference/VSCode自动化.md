# VSCode自动化

这页记录的是老师在这场会议里对 VS Code 自动化的演示和判断。

## 基本流程

老师演示的思路大致是：

1. 打开 VS Code
2. 打开指定项目目录
3. 打开 code / codex 一类扩展
4. 等扩展激活
5. 在输入框粘贴需求
6. 回车
7. 让它继续干活

## 为什么写详细指令更稳

这场里老师反复强调：

* 只写自然语言，模型可能误解
* 指令写得越细，成功率越高
* 最好把等待扩展激活、定位输入框、回车这些动作都写进去

## 一个常见问题

编码型模型一听到“写代码”，就可能直接下场改代码。

这会导致：

* 它不走你原本设计的 UI 流程
* 它不一定帮你先开网页或先开扩展
* 它可能直接在项目里动手

## 老师的经验判断

* VS Code 自动化能做
* 但不算特别丝滑
* 如果想更稳，最好尽量用官方 docs / scopes / 原生能力
* 纯靠截图坐标，长远来看不够理想

## 权限

老师明确提到：

* 如果权限给成完全允许
* 后面执行会连续很多

这意味着：

* 更省确认
* 风险也更高

## 这页最重要的一句话

VS Code 自动化的难点不在“能不能”，而在“怎么让它长期稳定、路径可控”。
