# 双云架构

老师这场课反复强调的“双云架构”，主要是指：

* Cloudflare
* AWS

## 为什么是这两个

老师的判断是：

* Cloudflare 很适合做入口层、边缘层、前端托管和轻量 serverless
* AWS 更适合承接数据中心、核心后端、安全、异步和数据库能力

## 在这套架构里的分工

### Cloudflare

* Waiting Room
* Pages
* Workers
* 边缘网络

### AWS

* WAF / Shield
* AppSync
* Lambda
* CloudWatch
* RDS / Aurora
* ElastiCache
* S3
* SQS / SNS
* ECS / ALB

## 为什么老师看重它

* 更贴近海外远程工作语境
* 更符合现在的弹性云和 Serverless 路线
* 能把前端、边缘层和核心后端分层得更清楚

## 这场课里更明确的理解方式

老师这场其实在逼大家把一条交付链记清楚：

* Cloudflare 先负责入口、边缘层和前端托管
* AWS 再负责核心服务、数据库、安全和异步
* 最后再由 Serverless / 弹性服务把整个系统串起来

所以双云架构在这里不是“多会两家云厂商”，而是：

> 学会把前端、边缘、后端、数据和安全放到一张图里理解。
