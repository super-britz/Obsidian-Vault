# 自动化测试

老师这场直播里讲的自动化测试，不只是传统意义上的测试用例，而是浏览器控制、性能分析和任务执行的组合。

## 他强调的能力

* 控制浏览器跑流程
* 自动点页面
* 收集性能数据
* 分析火焰图和 Long Task
* 根据 PRD 生成测试脚本
* 输出测试报告

## 直播里出现的关键连接

### 浏览器控制

老师认为浏览器能力是 AI 开发链路里非常关键的一环，因为很多流程都能直接在浏览器里完成验证。

### PRD -> 测试脚本

他很看重这一步：

* 先生成 PRD
* 再把 PRD 喂给自动化测试工具
* 让 AI 根据 PRD 生成测试脚本和报告

### 性能诊断

老师明显把性能分析和面试联系得很紧：

* 给你一个网址
* 让你指出最严重的问题
* 再给解决方案

## 这页的底层结论

自动化测试在这里不是一个独立环节，而是 AI 研发流程中的核心验证环节。
