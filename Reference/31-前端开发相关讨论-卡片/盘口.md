# 盘口

这张卡来自 [[31-前端开发相关讨论｜直播笔记整理]]。

[[盘口]] 又叫 [[订单簿]]，展示当前市场的买单、卖单和成交深度。老师说这是交易所前端面试的重灾区。

常见追问：

- 后端通过 [[WebSocket]] 高频推送数据，前端如何处理？
- 大量价格档位如何渲染？
- 行情暴涨暴跌时，UI 如何避免卡顿？
- 数据是逐条更新、批量更新，还是按价格档位合并？
- 组件如何封装？

可用方案：

- 用 [[Web Worker]] 处理重计算。
- 用 [[虚拟列表]] 减少 DOM 压力。
- 用节流、批处理、聚合减少渲染频率。
- 用 [[Rust]] / [[WebAssembly]] 承担部分计算。
- 把盘口数据结构、排序、合并、清理规则写清楚。

老师提醒：盘口不是普通表格，它是金融业务里的专业组件。
