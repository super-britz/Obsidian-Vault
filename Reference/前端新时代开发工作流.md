# 前端新时代开发工作流

这页是 [[AI开发主流程]] 在前端岗位里的具体化版本。

## 主链路

1. 有一个产品想法
2. 用 [[Figma Make 与 Figma]] 快速生成设计稿
3. 把设计稿放进正式设计协作与开发链路
4. 接入 [[MCP工具栈]]
5. 生成代码
6. 补齐 PRD
7. 跑 [[前端自动化测试]]
8. 进入 [[浏览器控制与远程模式]]
9. 再往后接部署、运维、PR、汇报

## 这条线和传统前端流程的差别

传统流程更像：

* 设计稿 -> 前端手工实现 -> 手工调试 -> 手工提测

现在更像：

* 创意 / 设计稿 -> AI 生成代码 -> PRD 驱动测试 -> 浏览器执行 -> 远程环境持续工作

## 这条线里人最重要的职责

* 定义目标
* 维护设计和实现一致性
* 决定哪些地方可以自动化
* 决定哪些结果能过验收
* 把住业务和架构边界
