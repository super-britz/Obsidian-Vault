# 前端自动化测试

这页是 [[自动化测试]] 在前端工作流里的落地版本。

## 老师强调的关键点

前端自动化测试现在最关键的不是背工具名，而是把这几样东西连起来：

* PRD
* 浏览器
* 页面功能
* 测试报告

## 为什么 PRD 不能省

老师虽然承认现在很多人会先起界面，但他仍然强调：

* PRD 本来应该先行
* 现在即便后置，也还是要补
* 因为没有 PRD，自动化测试就不知道功能边界

换句话说：

* 设计稿描述的是“长什么样”
* PRD 描述的是“要做什么”

## 他提到的测试类型

* E2E 测试
* 单元测试
* UI 对比测试
* 测试报告生成

## 浏览器为什么这么重要

老师反复强调：

* 自动化测试最后还是要落到浏览器执行面
* 浏览器负责点、看、收集结果
* AI 负责理解指令和组织执行

## 这场直播里的典型做法

把 PRD 交给测试工具或浏览器侧 agent，然后让它：

* 按功能说明跑流程
* 检查功能是否通过
* 输出测试报告

## 这页最重要的结论

在前端工作流里，PRD 是自动化测试的输入，浏览器是自动化测试的执行面。
