# IGW 与 NAT Gateway

这页对应老师那段最值得单独拆出来记的内容：**私有子网怎么出网**。

## 先记住问题

如果服务待在私有子网里：

* 外部通常直接进不来
* 它自己有时也出不去

但真实业务又常常需要：

* 调外部 API
* 发 webhook
* 访问 GitHub
* 访问部分公网资源

所以就会进入这几个概念：

* `Internet Gateway`
* `NAT Gateway`
* `Endpoint`

## `Internet Gateway`

老师这场课里最强调的点是：

* 它更适合 **双向公网访问**
* 常见于 `public subnet`
* 互联网可以访问它，它也可以访问互联网

## `NAT Gateway`

老师想让大家记住的是：

* 它更适合 **私网单向出网**
* 私有子网里的服务可以访问公网
* 公网不能主动反向访问它

## `Endpoint`

这场课里老师把它讲成：

* 更像私有访问 AWS 服务
* 不一定要经过公网

## 这三者最值得背的一句

* `IGW`：双向公网访问
* `NAT Gateway`：私网单向出网
* `Endpoint`：私有访问 AWS 服务

## 老师为什么说这块重要

因为一旦你的 Lambda / Node / 服务进入 VPC 和私有子网，这块就不再是“可选项”。

你不理解这些，就容易出现：

* 服务访问不了外部
* 外部访问不了服务
* 明明代码没问题，但系统就是不通

## 这页最重要的一句话

这不是死记 AWS 名词，而是在学“私网里的服务到底怎么和外部发生关系”。
