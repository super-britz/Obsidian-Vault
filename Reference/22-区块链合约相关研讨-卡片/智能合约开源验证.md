# 智能合约开源验证

这节课里，[[智能合约开源验证]] 被当成判断项目可信度的最低门槛之一。

老师给出的课堂结论：

- 合约至少应该能在 [[区块链浏览器]] 上看到源码
- 只看地址不够，要看 `Contract` 页面
- 部署到测试链后，也应该尽量做验证和开源

为什么重要：

- 能让别人检查提款、授权、所有权等逻辑
- 能降低“黑盒合约”骗人的空间
- 是团队协作和交付的基础动作
