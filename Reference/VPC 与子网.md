# VPC 与子网

这页对应老师在 AWS 那场课里反复讲的网络基础。

## `VPC` 是什么

* `VPC = Virtual Private Cloud`
* 可以理解成 AWS 里的一个虚拟私有网络
* 应用服务、数据库、缓存等资源，通常会被放在这个网络里

## `Subnet` 是什么

* 子网是 VPC 里的分区网络
* 一个 VPC 下面可以有多个子网
* 不同子网承担不同职责

老师这场课里常见的分法是：

* `public subnet`
* `private subnet`
* `database subnet`

## 老师为什么一直强调它

因为企业里的 Node / Lambda / 数据库，不会随便裸露在外网。

更常见的形态是：

* 对外入口交给 `API Gateway`
* 应用服务待在私有子网
* 数据库待在更隔离的数据库子网

## `Security Group`

老师在课里把它直接讲成“防火墙”，这个理解是够用的。

它的作用是：

* 控制资源间能不能互通
* 控制谁能访问谁

也就是说：

* 资源建好了
* 不等于网络就通了

## 这页最重要的关系

如果一个服务要访问数据库，往往要同时看：

* 它在哪个 `VPC`
* 它在哪个 `Subnet`
* 它挂了哪个 `Security Group`

## 这页最重要的一句话

云上“连不通”，很多时候不是代码问题，而是网络和权限模型没配对。
