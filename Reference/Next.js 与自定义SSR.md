# Next.js 与自定义SSR

这场课里老师的态度不是“抛弃 `Next.js`”，而是更现实的：

* 默认先用 `Next.js`
* 但当它成为瓶颈时，自定义 SSR 才有意义

老师认为 `Next.js` 的优势：

* 生态成熟
* 能力完整
* 大多数 SSR 需求都能覆盖

老师认为它的风险点：

* 冷启动重
* 平台绑定明显
* 自部署体验不总是友好
* 近两年安全问题让人警惕
* 老项目迁移成本可能极高

所以决策顺序应该是：

1. 先判断 `Next.js` 是否真的满足不了你
2. 再判断自定义 SSR 是否值得长期维护
3. 不是为了“酷”去自研，而是为了冷启动、安全、稳定性、迁移成本这些真实问题
