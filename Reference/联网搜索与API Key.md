# 联网搜索与API Key

这页记录的是这场会议后半段最核心的排障主题。

## 现场试过的路线

老师这场来回尝试了这些方案：

* Web search
* 浏览器直接访问网页
* GitHub 公用 API
* 社区 API
* Brave Search 一类搜索 key
* 模型自身的联网能力

## 失败的原因不是一个

这场会很适合提醒自己：

* 联网失败不等于模型不行
* 常见原因包括：
* 网关拦截
* 代理异常
* API key 没配
* 配置文件被改坏
* 免费额度限流
* 当前模型本身不支持

## 老师的 fallback 思路

当 web fetch 或 search 工具不顺时，老师给出的替代路线是：

* 不用 fetch
* 直接打开浏览器
* 从网页层完成搜索和访问

这是一个很现实的降级方案。

## 免费和付费的现实

老师的态度比较实用：

* 免费方案不是没有
* 但容易限流、失效、装不上
* 付费一般更稳
* 但不是所有人都值得一上来就到处买 key

## 一个重要结论

如果模型理论上能联网，但环境把它拦住了，它更像是在报告“门没开”，而不是在胡编。

## 这页最重要的一句话

联网搜索问题，本质上是模型能力、工具配置、网络环境和鉴权体系共同作用的结果。
