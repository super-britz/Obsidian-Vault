# Redis 与 BullMQ

这张卡来自 [[33-跨境金融及求职讨论｜直播笔记整理]]，并根据架构图补充。

[[Redis 与 BullMQ]] 在这套系统里属于队列和后台任务基础设施。

适合处理：

- provider webhook 后续异步任务。
- 通知发送。
- 状态轮询。
- 重试任务。
- 风控任务。
- 出入金状态同步。

它和 [[SQS]] 的关系可以理解为：BullMQ 更偏应用内部队列和 worker 编排，SQS 更偏云基础设施级消息队列。具体项目可以二选一，也可以分场景组合。
