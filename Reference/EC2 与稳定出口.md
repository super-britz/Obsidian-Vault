# EC2 与稳定出口

这页记录的是老师为什么反复强调 EC2。

## 老师最想解决的问题

不是单纯“部署一台机器”，而是：

* 需要一个稳定独享的出口
* 替代共享代理 / 共享住宅 IP
* 让美区工具、Google 家族和相关服务更稳定

## 老师的方案思路

* 买一台 EC2
* 配弹性 IP
* 配出网
* 得到一个稳定独享的出口

这场课里他说得更彻底一点：

* EC2 不只是部署机器
* 它同时还是很多海外工具的稳定网络底座
* 如果前面节前作业已经做了，后面很多工具会顺很多

## 他的说法为什么像“家庭 IP”

老师是用非常口语化的方式在说这件事。

更准确的理解是：

* EC2 + 弹性 IP = 稳定独享出口
* 它不等于真正的居民家庭宽带住宅 IP

但在很多实际场景里，它能解决的核心问题是一样的：

* 稳定
* 独享
* 不和别人混用

## 老师的实操偏好

* 更偏向美国区域
* 提到俄勒冈州这类免税州
* 认为这样在成本和使用体验上更合适

## 这场课里额外强调的细节

* AWS 学习账户通常就够起步
* EC2 本身未必贵
* 真正要花的是弹性 IP 和网络侧配置
* 一个月通常也就几刀到十刀左右
* 他把 EC2 口语化地讲成“家庭静态住址 IP”，核心是帮助大家理解“固定、独享、稳定”

老师这场还额外补了几层：

* EC2 + 弹性 IP 也可以被当作“稳定的美国出口”
* 这对 Claude / Gemini / Chrome 一类服务的可用性非常关键
* 后续还会继续讲 VPC、子网、安全组、NAT Gateway、Endpoint
* 也就是说，EC2 不是孤立的一台机器，而是后面整条云网络的入口

## 成本判断

老师的口径大致是：

* EC2 本身未必贵
* 主要是弹性 IP 和出网配置要花钱
* 总成本仍然可控

## 这页最重要的一句话

老师执着于 EC2，不是为了教你租机器，而是为了给你一个稳定独享的基础网络出口。
