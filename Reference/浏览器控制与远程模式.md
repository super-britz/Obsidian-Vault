# 浏览器控制与远程模式

这是这场直播最值钱的主题之一。

## 浏览器控制

老师整场都在强调，浏览器现在不只是给人看的页面，而是 AI 的一个执行入口。

浏览器侧可以做的事情包括：

* 打开页面
* 搜索和导航
* 分析 Console / Network / 页面问题
* 执行自动化测试
* 根据指令操作已打开的网站

这场多事项讨论会议里，又补了几件很具体的事：

* 有状态浏览器和空浏览器完全不是一回事
* 某些版本必须把控制页和目标页放在同一组里
* 本地版本和线上版本可能不一致
* Mac 客户端支持通常比 Windows 更顺

## 默认浏览器和登录态浏览器的区别

老师特别提醒：

* 很多工具默认会打开一个新的浏览器环境
* 没登录，所以很多动作做不了
* 如果你想让 AI 操作“已经登录的账号”，通常需要额外插件或授权桥接

这意味着：

* 功能更强
* 风险也更高

## 本地模式

特点：

* 在本机工作
* 用本机目录
* 用本机浏览器
* 更适合学习和调试

## 远程模式

老师展示的远程模式，大致是：

* 连接仓库
* 自动创建远程环境
* 在远程主机上工作
* 自动开发
* 自动提交 PR
* 自动 review
* 最后由人确认

在多事项讨论会议里，这条链路还补充到了：

* 启动服务
* 截图
* 创建 PR
* 在 PR 下直接看到 review 结果

这说明远程模式已经不只是“写代码”，而是逐步接进了协作流程。

## 远程模式的真正变化

它改变的不是“代码写在哪”，而是：

* 工作环境从你本机迁到云端
* 你从执行者变成指挥者
* 你主要负责验收和决策

## 实际风险点

老师后半段专门提醒：

* 插件要验真
* 登录态浏览器要谨慎授权
* 别把财产相关账户接进这套环境
