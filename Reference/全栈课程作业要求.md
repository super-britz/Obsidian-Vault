# 全栈课程作业要求

这场全栈课的作业，已经从“体验工具”升级到“跑通工程链路”。

## 旧作业

老师提到前面的作业主要是：

* 根据自己的想法生成设计稿
* 通过 export 或 MCP 导入 Figma
* 把设计稿接进开发链路

## 这次的新作业

### Cloudflare 侧

* 注册 Cloudflare
* 买一个域名
* 把域名绑定到 Cloudflare
* 给 React 项目做 CI/CD

### 前端项目

* 用 React 起一个项目
* 老师建议直接用 `Vite`

### Workers 侧

* 部署一个 Cloudflare Worker
* 用 `export default` 方式起服务
* 让 Worker 去请求 DeepSeek 或 OpenAI

### 接口层

* GraphQL
* tRPC
* 以及部分上下文里提到的其他接口协议

更稳妥的理解是：

* GraphQL 长期必须补
* 这场转写里对接口层有一些混用和口误
* 当前更重要的是先把 Worker + 接口层 + MCP 一起跑通

### MCP 侧

* 选 1 到 2 个 MCP
* 让它解决一个实际问题

## 提交物

* 域名
* 链接
* 项目结果

## 这场课里补出来的交付细节

老师把“交什么”说得更像真实工程提交物：

* GitHub 地址
* 项目文档
* 自动化测试结果
* 测试报告
* 中途遇到的问题和心得

另外他还反复提到：

* 最基础的 AWS 学习账户要准备好
* EC2 至少要有基础认知
* 自动化是重要方向，但这轮不一定把“全自动”当硬性门槛

所以这次作业更像是：

> 先把手工工程链跑通，再逐步把其中可重复的部分自动化。

## 老师真正想看什么

不是“页面做得多花哨”，而是你有没有把这条链路真正跑通：

* 域名
* Cloudflare
* React
* Worker
* 接口
* MCP

## 更贴近这场课的理解

这次作业的真正目标不是“把功能糊出来”，而是开始让大家熟悉：

* 设计稿之后的工程接法
* Cloudflare 的前端托管和 Worker
* AWS / EC2 这类基础云概念
* MCP 在真实场景里的用法
* 交付时要不要留文档、测试和复盘
* 不要因为某个协议名卡住整条作业链
