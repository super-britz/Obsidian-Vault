# 前端技术雷达

这页对应老师在“技术选型”课里反复刷榜单和趋势页的那种训练方式。

## 老师为什么一直看这类页面

不是为了“追新”，而是为了训练：

* 看趋势
* 看成熟度
* 看生态
* 看社区
* 看项目和团队是否适配

## 这类雷达里，老师这场重点盯的类别

### 前端框架

* `React`
* `HTMX`
* `Svelte`
* `Solid`
* 以及其他新一代轻量框架

### UI 和设计

* UI 组件库
* 设计到代码工具
* 原型工具

### 数据和认证

* `Supabase`
* 授权 / 认证框架

### 浏览器自动化

* `Playwright`
* `Puppeteer`
* 以及更偏 AI 的浏览器控制工具

### 包管理和工程化

* `PNPM`
* `monorepo`
* 构建工具

### 移动端

* `React Native`
* `Flutter`
* 新一代更轻、更双线程的移动端方案

## 老师真正想训练的能力

不是“记住榜单”，而是能回答：

* 为什么这个东西会火
* 为什么它不一定适合当前项目
* 它会给团队带来什么收益
* 它可能引入什么维护成本

## 这页最重要的一句话

技术雷达不是让你盲目追热度，而是让你学会“把趋势翻译成项目决策”。
