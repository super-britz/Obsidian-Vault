# 编程课程筹备事宜｜直播笔记整理

## 概览

这不是一节正式技术课，而是一节 [[课程定位|开营说明 + 行业判断 + 工具准备 + 学习路径 + 求职预告]] 的综合说明会。

老师更强调的是：

1. 行业环境变了
2. 这门课偏结果导向，不只是“学点东西”，而是为了更好找工作、做项目、提高收入
3. 开课前需要先准备好 [[开课前准备]]，否则后面会持续掉坑里

## 老师对学员现状的判断

老师反复提到，现在来上课的人和前些年不一样：

* 以前很多同学是“本来发展不错，再来提升一下”
* 这几年越来越多的人，是已经遇到职业瓶颈、人生瓶颈，甚至比较焦虑，不知道下一步怎么走
* 所以这几期课他带得更累，因为很多人带着强烈期待来，不只是为了兴趣，而是希望通过课程改变收入和职业状态

> [!note]
> 这决定了整门课的风格：不是纯知识型，而是偏“实用、硬核、能落地、能变现、能求职”。

## [[求职判断]]

### 行业阶段变化

老师把行业大致分成几个阶段：

* **2016 ~ 2019**：Web2 最后一波大潮，整个互联网环境好
* **2021 ~ 2023 左右**：Web3 风口期，很多公司都在招人
* **2024 年后**：Web3 公司数量明显减少，没有前几年那么容易进了

### 更现实的机会方向

老师给出的判断是：

* 年轻、学历和背景还可以的人，仍然建议先争取北京大厂
* 如果年龄、学历、背景、技术水平够不上大厂，不要硬碰
* 现在更现实的机会主要是：
  * 北京的 AI 创业公司
  * 深圳 / 靠近香港的 Web3 相关线下机会
* 远程机会没有以前那么多了

> [!summary]
> 老师对求职方向的现实判断：
> 第一梯队：北京大厂
> 第二梯队：北京 AI 创业公司
> 第三梯队：深圳 / 香港附近的 Web3 / 出海类岗位

## [[开课前准备]]

这一部分是整场直播最核心、最具体的内容。

### [[设备与账号安全]]

#### 设备准备

老师强烈建议学员准备一台单独的二手 iPhone，最好 iPhone X 及以上。主要原因包括：

* 后续有很多支付、工具安装、账号登录场景
* iPhone X 及以上具备面容 / 生物识别，更适合这些流程
* 后续涉及“钱”的东西尽量都放在单独设备里操作

老师原话里对安卓很不信任，提到有同学因为安卓设备问题丢过钱，还提到剪贴板被劫持之类的风险。

> [!warning]
> 这里要区分“老师的强烈主张”和“客观结论”：
> 老师的主张：尽量不要用安卓做这些敏感操作。
> 更稳妥的客观结论：重要账号、资金、支付、工具订阅，需要设备隔离和安全管理。
> 核心不是“安卓一定不行”，而是不要把高风险操作放在不受控设备上。

#### Apple ID

老师要求准备两个 Apple ID：

* 港区 Apple ID
* 美区 Apple ID

原因是不同软件在不同地区商店的可用性不同。

他同时强调：

* 不要去淘宝买账号
* 尽量自己注册
* 如果买了别人的 Apple ID，也不要直接登录系统设置，避免设备被锁

> [!danger]
> 建议记成：
> 不要买来路不明的 Apple ID。
> 不要在系统层登录别人的 Apple ID。
> 只用自己可控、可找回、可验证的账号体系。

### [[手机号与身份体系]]

老师建议把手机号拆成几类：

* 香港手机号：用于 Telegram、部分海外服务、某些非大陆手机号更方便的场景
* 虚拟手机号：用于部分 AI 工具、非核心账号的辅助注册
* 大陆手机号：用于涉及资金、找回和安全的重要账号场景

> [!tip]
> 这一段真正想表达的底层逻辑是：不同账号、不同用途，不要混在一个手机号体系里。

> [!warning]
> “虚拟手机号”和某些非实名方案有明显风险。更稳妥的替代方式是：
> 核心账号只用自己实名、长期可控的手机号。
> 非核心账号也尽量别依赖来路不明的号码源。

### [[支付与收款体系]]

老师很重视海外工具付费能力和后续收款能力，所以讲了大量关于交易所、稳定币、卡片、收款路径的内容。

#### 老师想解决的问题

* 海外 AI 工具没法付费
* 不知道怎么给海外服务充值
* 后续就算找到工作，也不知道怎么收钱、怎么消费、怎么出金

#### 他的方案思路

直播里他的思路是：

* 准备交易所
* 准备可用于海外支付的卡
* 通过稳定币和卡去完成工具付费
* 后续工资、项目款也可能通过这类体系流转

#### 提到的具体要素

* 币安
* USDT
* 美卡 / 蓝卡 / 紫卡
* 香港中银、汇丰等香港银行卡
* 大陆银行卡
* 护照、港澳通行证

> [!warning]
> 不要把它记成“必须立刻照做的操作清单”。
> 更合适的理解是：老师想搭一个出海支付 / 订阅 / 收款 / 消费闭环。
> 但里面有不少环节涉及合规、账号安全、资金风险。
> 正式执行时，应该优先选择官方、合规、可追溯的支付和收款方式。

### [[证件与出行准备]]

老师认为以下也是后续课程和就业的基础准备：

* 护照
* 港澳通行证
* 香港银行卡
* 某些跨境场景下的身份准备

他的意思很明确：

* 不要总局限在纯中文环境里
* 要尽早为出海工作、跨境工具、英文环境做准备
* “走出去”不只是旅游，而是为后面职业路径做准备

### [[网络环境要求]]

老师对“网络环境干净度”非常在意，认为这直接影响：

* 账号是否容易被封
* 工具能否正常注册和使用
* 海外服务的稳定性

他反复强调：

* 代理必须“干净”
* 不干净就容易频繁封号
* 还提到检测 IP 风险值、原生 IP、住宅 IP 等说法

> [!warning]
> 不要直接照着直播口述去操作。
> 你可以在笔记里记成：
> 老师认为网络环境会影响海外服务可用性。
> 他非常推崇“干净 IP / 原生 IP / 住宅 IP”。
> 但实际执行时，要以官方可用、合规、安全为优先，不要走高风险灰色路径。

## [[AI工具栈]]

### 整体理解

老师认为现在成熟的 AI 产品，基本都在形成一套类似的结构：

1. 聊天客户端
2. 命令行 CLI
3. IDE / VS Code 插件
4. 浏览器能力
5. 模型调用 / agent 能力

也就是说，他不是把 AI 当成单个聊天工具，而是看成一套完整工作平台。

### 重点提到的工具

#### Claude / Claude Code

老师对 Claude 的评价非常高，几乎把它当成当前编程主力。相关描述包括：

* 很适合写代码
* 有客户端、命令行、插件
* 后面做 agents 可能还要买模型能力
* 支持一种“远程虚拟机 / remote”工作模式
* 可以实现“一个 AI 看着另一个 AI 工作”这类自动化思路

#### ChatGPT / OpenAI

老师提到它的特点更偏“全家桶”：

* 聊天
* 浏览器
* 图片
* Sora 视频
* 某些客户端能力

#### Gemini

老师认为它：

* UI 生成能力很强
* 浏览器入口很强
* 也有 CLI
* 还有一个官方 IDE，被他称为“反重力 IDE”

#### 本地模型

老师要求大家最好准备一个本地模型兜底体系，例如：

* Ollama
* 千问等本地可跑模型

他的理由是：

* 在线模型会受限
* 本地模型可以做保底
* 有时不需要付费也能完成一部分任务

### 使用场景

#### 自动化测试与浏览器操作

老师很看重浏览器能力，认为它可以：

* 做自动化测试
* 帮你操作复杂后台
* 在你不熟悉平台页面时，替你点按钮、找配置项

#### 产品设计图 / UI 生成

他提到后面实战作业可能要生成漂亮的产品设计图，因此会用到擅长 UI 生成的模型。

#### 代码迁移 / 重构 / 跨语言改写

老师举的案例里提到：

* Python + Golang 微服务逻辑迁移到 Java
* 借助 AI 大幅缩短开发周期

#### 企业级 AI 流程改造

老师声称自己在做企业培训时，已经把“从想法到上线”的流程几乎全部 AI 化。

> [!summary]
> 他想传达的不是“AI 只是帮你补代码”，而是：
> AI 可以进入产品、设计、测试、部署、运维、求职、英语、自动化等全链路。

## [[开发环境要求]]

老师对硬件和系统的要求比较明确：

* 电脑最好是 Mac，最好 32G 内存以上。16G 也能凑合跑，但不理想。
* 经费有限可以考虑 Mac mini，再配一个便携显示器。
* 实在不行就 Linux。

老师强烈不建议把 Windows 作为主开发环境，理由是：

* shell 脚本体验差
* 某些 CLI / 客户端容易出问题
* 跑模型可以，但基于模型继续开发不顺

> [!warning]
> 更客观的替代说法：
> Mac / Linux 更适合这类 AI 开发工作流。
> Windows 不是不能做，但需要额外处理 WSL2、终端、依赖环境等问题。
> 真正重点是：终端能力、shell 能力、CLI 工作流能力。

## [[课程路线图]]

老师把后续课程分成大致五个阶段：

### 阶段 1：先会用 AI

* 先把工具付费、安装、聊天、基础使用弄顺
* 先能正常用起来

### 阶段 2：深入用 AI

* MCP
* skills
* slash commands
* 从产品、设计、测试、后端、运维等角度看 AI 怎么用

### 阶段 3：AI 控制 AI

* 让一个 AI 看着另一个 AI 工作
* 自动化 AI 工作流

### 阶段 4：基于 AI 的开发

* agents
* RAG
* 模型微调

### 阶段 5：AIGC / 视频内容方向

* 生成图片
* 生成视频
* 老师口中说的是 “AIDC”，但大概率想表达的是 AIGC 方向

## [[AI使用原则]]

> [!important]
> 你的技术体系，一定要凌驾于 AI 之上。

老师反复强调：

* AI 会生成代码
* AI 会给架构
* AI 会给方案
* 但它可能是旧方案、错方案、不合适的方案
* 如果你自己没有判断力，就会被 AI 带偏、绑架

### 这条原则可以展开成四句话

1. AI 很强，但不是最终裁判
2. 人必须 review
3. 技术判断必须在人手里
4. 会用 AI 的前提，是自己知道“对错、好坏、适不适合”

### 老师对“会用 AI”的真正定义

在老师这里，“会用 AI”不是指会问问题，而是：

* 能明确目标
* 能拆任务
* 能看出 AI 生成结果的问题
* 能二次下指令修正
* 能最终落地成你想要的结果

> [!summary]
> 在他的理解里，AI 用到高阶阶段，不是“让 AI 替我想”，而是“我比 AI 更清楚目标，让它高效执行”。

## [[英语路线]]

老师明确认为：

* 纯中文环境更卷
* 英文能力一上来，工作机会会明显增加
* 英语好 + 技术好，工作很好找

### 他的英语观不是应试观

他反对把精力放在：

* 先报雅思班
* 先考证
* 先纠结标准口音

他主张的是：

* 先敢开口
* 先能表达
* 先会自我介绍
* 先会讲简历、讲项目
* 先能进行工作面试沟通

### 他的练习路线

老师提议：

* 先去香港
* 再去新加坡、马来西亚、泰国等地
* 用真实生活场景逼自己张嘴
* 回来再背行业词汇
* 再结合 AI 做面试口语练习
* 从 `introduce yourself` 开始练到能讲完整简历

### 对“听不懂”的看法

老师认为现在不用过度担心：

* 有 AI
* 有翻译软件
* 有硬件设备
* 听不懂可以辅助
* 真正难的是“不敢开口”

> [!tip]
> 这一段最值得保留的结论是：
> 面试英语不等于考试英语。先解决开口表达，再解决精度问题。

## [[面试变化]]

老师提到，现在面试也在变：

### AI coding tools

面试官会问：

* 你用过哪些生成式 AI coding tools
* 你喜欢它们什么
* 不喜欢什么
* 它们各自的优缺点是什么

### 真实问题诊断

他举的例子是：

* 给你一个平台
* 让你找出最严重的 5 个问题
* 再给出解决思路

老师认为，如果你会借助 AI 做快速诊断，并且输出准确、有洞察，就会非常加分。

> [!important]
> 这说明老师课程里很想培养的一种能力是：
> AI 辅助下的快速分析、快速定位、快速输出能力。

## 课程节奏与班级要求

直播里也交代了课程运作方式：

* 开营这两天先准备东西，不正式上课
* 后面周六周日上午上课
* 前期准备项很多，不是一天就能全搞定
* 后面会开始刷题：
  * 一天一道算法题
  * 晚上发讲解
  * 之后还有 TS、手写等内容

老师还提到群内沟通注意事项：

* 某些关键词不要频繁直接发
* Telegram 会作为重要沟通工具
* 有问题可以在群里交流或找班主任反馈

## [[风险与替代方案]]

### 疑似转写错误词汇表

以下是根据上下文推测的高概率纠错，后续正式笔记建议统一替换：

* `cloud / core / 克劳德` -> **Claude / Claude Code**
* `codis` -> **Codex**（也可能是别的 CLI 名，但大概率是 Codex）
* `精米 / 机密 / g mini` -> **Gemini**
* `反重力 IDE` -> **Antigravity**
* `欧拉玛` -> **Ollama**
* `AIDC` -> **大概率是 AIGC**
* `t` -> **Telegram**
* `BI` -> **大概率是 Binance / 币安**
* `HS ACACAC` -> **HSBC**
* `account has been disable` -> **account has been disabled**
* `阿特拉斯 / 阿法斯` -> **Atlas**（但不完全确定）
* `seework` -> **大概率是 CoWork / computer-use 类能力，不确定**
* `block` -> **可能是 Grok，也可能是别的工具，不确定**
* `XTP` -> **大概率是 GPT / ChatGPT**
* `hero message service` -> 某个短信平台，**不确定具体名称**
* `let Chrome Bros…` -> 某个 Chrome 浏览器操作能力相关入口，**转写严重失真**

### 不该照抄照做的部分

以下内容可以作为“老师的个人经验观点”记录，但不建议直接当执行手册：

* 买号 / 成品号 / 借身份
* 来路不明的 Apple ID、学生号、特殊身份号
* 把代理、支付、交易所当成所有人的第一优先级
* “安卓一定不行”“Windows 一定不行”

更客观的说法是：

* 重要的是安全隔离、系统稳定性、终端能力、开发环境控制能力
* Mac / Linux 更省心，但不是唯一道路

## 顶部总结

> [!abstract]
> 这门课的核心不是“教一个工具”，而是：
> 先搭一套 AI 时代的工作环境。
> 再建立 AI + 编程 + 求职 + 英语 + 自动化 的整体系统。
> 最后把它用到工作、项目、面试和赚钱上。
> 但前提永远是：**人要比 AI 更懂目标、更懂技术、更懂取舍。**
