## 一、脑图给出的课程主线

### 0. 起步招式是 Web2

脑图顶部直接写了：

* `0. 起步招式 - Web2`

这和老师口头讲的是一致的：

* Web3 前端开发底座还是 Web2
* 主力技术栈仍然是 React
* 课程不围绕 Vue 展开
* 找工作时，海外和 Web3 语境里 React 更常见

所以老师真正想表达的是：

> Web3 不是绕开 Web2 重新学一套，而是在 Web2 基础上加钱包、签名、RPC、合约交互。

### 1. 走进区块链的基本准备工作

脑图把这一层拆得很清楚：

* 行业知识
* 常见公链
* 水龙头

其中行业知识部分写了：

* `CoinMarketCap`
* 面试前准备
* 币安 / OKX 标签
* 一定了解清楚

这和老师直播里的意思完全一致：

* 找工作前，不懂行业标签不行
* 面试一定会问你公司所在赛道、竞品和行业位置

常见公链部分，脑图也直接把开发主战场写死了：

* `Ethereum`
* `Solana`

并且标了对应的测试网准备：

* 以太坊的测试水龙头
* Circle 的测试 USDC
* Solana faucet
* `SOL` 作为手续费

这一点正好帮直播稿校正了一个核心事实：

> 这节课不是泛讲所有链，而是先围绕 **EVM + Solana** 两条主线准备环境。

### 2. 钱包调用封装

脑图这一层非常关键，因为它把老师后半段口头提到的各种库真正归位了。

#### Ethereum 侧

脑图列出的主流路线是：

* `web3-react`
* `viem`
* `wagmi`
* `RainbowKit`
* `ConnectKit`

其中关系大致可以理解成：

* `viem` 更偏底层引擎
* `wagmi` 在 React 里很常见
* `RainbowKit` / `ConnectKit` 更偏钱包连接 UI 套件

#### Solana 侧

脑图列出的路线包括：

* `@solana/kit`
* `@solana/wallet-adapter-react`
* `@jup-ag/wallet-adapter`
* `solana-foundation/connectkit`
* `Privy`
* `solanakit`
* `@solana/web3.js`
* `gill`

脑图同时还写出了一条很重要的话：

* `提供一个可以操作钱包的 provider -> 签名`

这其实就是老师一直在说的：

* 先连接钱包
* 拿到 provider
* 触发签名
* 才能继续做转账和链上操作

### 3. 和合约交互

脑图把这部分整理成两层：

#### 第一层：RPC 节点

它明确写了：

* 三方平台
* RPC 节点
* Ethereum
* Solanan
* TPS
* QPS

这和老师直播里的意思一致：

* 合约交互要先有 RPC
* RPC 可以是官方、自建或第三方
* 链也会有拥堵、性能和 TPS 问题

#### 第二层：ethers.js 技术体系

脑图把老师讲的 `ethers.js` 重点整理成了一串清单：

* 包装 Provider
* 读取 view 合约内容
* 发送 ETH
* 调用合约
* 部署合约与 ABI / TypeScript
* 检索事件
* 事件过滤
* `bignumber` 单位转换 `wei / gwei`
* `callStatic`
* 识别 ERC721
* 编解 `calldata`
* 批量生成钱包
* 批量转账
* 批量归集
* `MerkleTree` 脚本
* 数字签名脚本
* 监听 `Mempool`
* 解码交易数据

脑图左侧还补了一些应用场景：

* `ethers.Wallet.createRandom()`
* 交易发起前先检查会不会失败
* 白名单铸造 NFT
* 多钱包归集管理
* `HDNode`
* `BIP32 / BIP44 / BIP39`
* 获取合约内部 interface 做更底层操作

这部分特别有用，因为直播里老师很多术语都口误成了：

* `EOS`
* `EOUS`
* `easters`

脑图把它校正回来了：

> 这里大部分时候，他说的其实就是 **ethers.js**。

### 4. 学习合约

脑图最后收得很干净：

也就是：

* 环境准备
* 钱包接入

* 然后进入合约

这和老师最后口头预告完全一致：

* 下周开始讲合约
* 然后再讲 Web3 和 Web2 架构融合

## 二、老师这场课补了哪些赛道

这场直播比上一节更像赛道地图课，重点补了这些方向：

* [[Web3赛道地图]]
* [[RWA]]
* [[NFT 与数字藏品]]
* [[Prediction Market]]

### 1. DePIN

老师在讲：

* 贡献硬盘
* 贡献网络
* 贡献设备
* 拿代币

高概率是在说：

* `DePIN`
* `Filecoin / FIL`

### 2. RWA

老师重点强调：

* 房子
* 国债
* 应收账款
* 现实资产拆分后链上流通

并反复说：

* 面试做 RWA 的公司，几乎一定会问你怎么看 RWA

### 3. NFT 与数字藏品

这场课把 NFT 分成了两类：

* 国外的纯血 NFT
* 国内的伪 NFT / 数字藏品 / 实物绑定凭证

老师认为：

* 真正给大多数人带来工作机会的，往往不是纯血 NFT
* 而是和现实资产、版权、鞋、活动资格绑定的数字凭证

### 4. Web3 + AI

老师提了两个典型交叉方向：

* 聚合显卡算力 / 渲染网络
* 给 agent 发币的平台

高概率对应：

* `RNDR / Render`
* `Virtuals`

### 5. 元宇宙 / GameFi / Move-to-Earn

这场课把这些也扫了一遍：

* 元宇宙地皮
* 对战养成类 GameFi
* 走路赚钱

他的目的不是让大家去炒，而是强调：

* 面试可能问
* 行业知识得知道

### 6. Prediction Market

这是老师认为“今年更火”的方向之一。

而且他用了自己面试过的真实案例来说明：

* 新赛道面试会直接考产品理解
* 会问你竞品、交互、概率、价格区间、用户互动、前端问题定位

所以这块我单独拆成了 [[Prediction Market]]。

## 三、老师对面试的核心判断

这场课里，老师对面试的判断有三层。

### 1. 现在会直接考产品理解

不是只考八股，而是直接给你：

* 产品链接
* Google Form
* 竞品
* 实际页面

让你回答：

* 这个赛道解决什么问题
* 平台最严重的 5 个问题是什么
* 怎么改
* 前端工程师应具备什么能力
* 你怎么使用 AI coding tools

### 2. 不玩产品就很难过

老师的态度很明确：

* 你不一定今天就深挖
* 但在面试前，目标公司所属赛道和竞品一定要玩

否则他会觉得：

* 你根本不懂他们在做什么

### 3. 这类题不仅考技术，也考情商

老师专门提醒：

* 你不能把对方产品批得一无是处
* 也不能答得太浅
* 要说出问题，同时又别让对方下不来台

## 四、老师给出的开发主线

把直播稿和脑图合起来，老师真正要让大家建立的是这条线：

1. 以 Web2 为起点
2. 先理解赛道和公链
3. 准备钱包和测试网
4. 学会钱包连接、provider、签名、转账
5. 再进入 RPC 和合约交互
6. 然后学合约

也就是说：

> 这节课其实就是“合约课之前的环境与接入准备课”。

## 五、老师本周布置的作业

这部分是整场最实操的内容。

### 作业目标

跑通最小 Web3 前端开发链路。

### 作业内容

* 注册并准备两个钱包
* 保留好助记词
* 领取测试网水龙头
* 做一个页面
* 页面至少有按钮和文本框
* 完成一次 ETH 转账
* 完成一次 Solana 转账

### 老师强调的作业提交物

* 代码讲解
* 代码仓库
* 项目链接

### 他对作业的态度

老师整场都在强调：

* 不交作业，最后基本学不会
* 听懂不等于会做
* 真找工作时，能不能自己跑通这条链才重要

## 六、这场课里最值得保留的结论

> 这节课的真正价值，不只是补几个 Web3 概念，而是把路线收成了一张很明确的图：
>
> * Web2 是起步
> * 赛道认知是面试前提
> * 以太坊和 Solana 是当前主战场
> * 钱包、测试网、水龙头、provider、签名、转账是第一道门槛
> * 过了这一步，下一周才会真正进入合约
