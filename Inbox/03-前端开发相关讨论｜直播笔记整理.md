# 前端开发相关讨论｜直播笔记整理

## 概览

这节课是“前端新时代开发工作流”的第二节，重点不是讲某个框架，而是在讲一套完整的 [[前端新时代开发工作流]]。

老师想传达的核心是：

1. AI 已经开始重构前端工作流
2. 前端不再只是“切页面”，而是要会把设计稿、MCP、浏览器、测试、远程环境串起来
3. 人的角色正在从手工实现者，变成流程指挥者、工具调度者和结果审查者

## 核心结论

这场直播最值得保留的结论有几条：

* 前端开发已经从“手工实现页面”转向“AI 驱动的整条交付链路”
* 设计稿、浏览器、MCP、PRD 和自动化测试正在成为一个系统
* 浏览器已经不只是调试工具，而是执行和验证的核心入口
* 自动化测试不是新东西，但 AI 把它从“脚本工程”变成了“自然语言可调度能力”
* 真正不能丢的，仍然是开发规范、设计一致性、结果验收和安全边界

## [[前端新时代开发工作流]]

老师这一场的主线，大概可以整理成：

1. 从想法出发
2. 用 [[Figma Make 与 Figma]] 生成设计稿
3. 把设计稿放进正式设计协作和开发链路
4. 接入 [[MCP工具栈]]
5. 生成代码
6. 根据 PRD 跑 [[前端自动化测试]]
7. 进入 [[浏览器控制与远程模式]]
8. 再往后接部署、远程运维、PR、汇报等协作动作

老师的意思不是“AI 帮你省一点时间”，而是：

> 前端已经进入从设计到交付都可以被 AI 贯穿的阶段。

## [[Figma Make 与 Figma]]

老师特别强调大家不要把这两个东西混在一起。

### Figma Make

* 更像一个 AI 聊天入口
* 适合从想法快速生成设计稿
* 重点是“起稿快”

### Figma

* 是正式的设计软件
* 负责团队协作、编辑、承接开发
* MCP 和后续开发衔接发生在这里

### 正确顺序

老师讲的顺序是：

* 先用 `Figma Make` 起草设计
* 再把结果导进 `Figma`
* 然后再接开发链路

### 进一步补充

老师还提到一些和视觉生成有关的东西：

* `AI Studio` 更像综合控制台，不只是普通聊天网页
* `Nano Banana` 一类图像能力可以用来做“垫图”
* “垫图”就是先生成高质量风格参考图，再反过来约束后续设计结果

## [[AI工具栈]]

这场直播里，老师实际上在讲一套前端侧的 AI 三件套甚至四件套：

1. IDE / 插件
2. CLI
3. 浏览器入口
4. 设计稿 / MCP 接口

他明显更偏向的第一梯队是：

* Claude
* Gemini
* Codex

第二梯队则包括：

* Cursor
* Trae
* 千问
* 豆包等国产模型

他的口径很强，但可以记成：

* 第一梯队更适合作为主力生产工具
* 第二梯队可以用，但整体体验和上限差一些

## 浏览器入口为什么这么重要

老师整场课都在强调浏览器，不只是因为前端在浏览器里跑，而是因为：

* 浏览器现在是调试入口
* 也是自动化测试入口
* 还是很多 AI 工具真正的执行入口

他很看重 Google / Chrome / Gemini 这一线，一个核心原因就是：

* 浏览器整合能力更强
* Console / DevTools / 右上角对话入口能直接接进工作流

## [[前端自动化测试]]

老师认为前端自动化测试现在最重要的不是“记住几个框架名字”，而是理解这条链：

* PRD 说明产品要做什么
* 浏览器负责执行和观察
* AI 负责根据 PRD 跑功能流程
* 最后输出测试报告

他提到的测试形态包括：

* E2E 测试
* 单元测试
* UI 对比测试
* 测试报告生成

### 一个关键点

老师特别强调：

* 设计稿可以先出
* 但 PRD 仍然不能省
* 因为没有 PRD，自动化测试就不知道应该测什么

所以这场课里一个很重要的结论是：

> PRD 在 AI 工作流里不只是产品文档，还是测试输入。

## 设计稿和开发稿必须保持一致

老师虽然整场都在讲 AI 提效，但也反复提醒：

* 生成代码以后，开发可能改了很多
* 如果设计稿没同步，就会出问题
* 设计和实现一致性，仍然是前端基本功

这个提醒很朴素，但非常重要：

> AI 可以加速产出，不能替你负责一致性。

## [[浏览器控制与远程模式]]

这一段是这场直播里最值钱的部分之一。

老师讲了两种工作方式：

### 本地模式

* 在你本机开浏览器
* 在你本机代码目录里工作
* 用本机执行和验证

### 远程模式

* 在远程环境中工作
* 直接连仓库
* 自动创建环境
* 自动开发
* 自动提交 PR
* 自动 review
* 人最后只做确认和验收

他想表达的是：

> AI 已经不只是“帮你写一段代码”，而是在一台远程机器上替你持续干活。

## [[模型授权与成本]]

这场直播里，老师给了很多现实建议：

* Claude 仍然是最强主力之一
* 但贵，而且限制多
* 相比纯 API key，他更偏向网页授权 / Web Auth
* GitHub Copilot 可以作为一种低成本迂回路径
* 本地大模型目前不适合作为大多数人的主力方案
* Apple Silicon 的实际体验明显更好

## [[AI工作机安全边界]]

老师后半段非常强调安全，这一块不要忽略。

他的底层意思是：

* 跑 agent 的机器不要和个人主身份混在一起
* 不要登录财产相关账户
* 不要乱装扩展、插件、security 工具
* 不要轻易暴露可被接管的账号入口

这一页和 [[设备与账号安全]] 是强相关的。

## 课程外延：学习、表达与低成本创业

这场直播不只是在讲“前端编码”，还反复展示 AI 还能做：

* PPT
* 思维导图
* 报告
* 知识卡片
* 视频转内容

老师想传达的其实是：

* 学习
* 汇报
* 内容输出
* 低成本创业

都已经被 AI 重构了。

## 可执行版总结

如果把这场直播压缩成最值得执行的版本，可以记成：

1. 先把 `Figma Make -> Figma -> MCP -> 代码 -> PRD -> 浏览器测试` 这条链跑通
2. 先学会本地模式，再理解远程模式
3. 正式把浏览器交给 AI 前，先建立 [[AI工作机安全边界]]
4. 不要只记工具名，要记住每一步的输入、输出和验收点
5. 保持设计稿、实现、测试和 PRD 之间的一致性
