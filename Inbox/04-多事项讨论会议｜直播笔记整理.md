# 04-多事项讨论会议｜直播笔记整理

## 概览

这场直播不是按章节推进的正式课程，更像一场高密度的实操答疑会，主要在做几件事：

1. 现场切模型、切扩展、切联网方式
2. 演示浏览器、APP、VS Code 的自动化操作
3. 讲怎么把零散能力收束成 [[agent军团]]
4. 排查为什么有的模型能联网、有的模型不能联网
5. 给下周的全栈和代码课程做过渡

这场会里最值得保留的主线，不是某个单一工具，而是：

* 你怎么选模型
* 你怎么装可控扩展
* 你怎么让浏览器、APP、VS Code 被 AI 调度
* 你怎么把这些能力最终组织成“离开电脑还能跑”的工作流

## 这场会最核心的结论

老师反复在讲几件事：

* 模型差异非常大，不能把所有模型当成一个水平
* 浏览器自动化、APP 自动化、VS Code 自动化，本质上不是一回事
* 你真正想做的不是“少点几下按钮”，而是离开电脑后它还能继续干活
* 最后有价值的不是某次演示，而是多个 agent 组成一个“虚拟办公室”
* 现在很多失败不是“你不会用”，而是模型、版本、权限、网络、key、网关这些基础层没配好

## [[模型与扩展选择]]

这场会里，老师把模型和扩展的判断说得很直白：

* 日常默认模型能凑合用
* 真想特别顺，`Claude` 仍然是主力
* 编码型模型一听到“写代码”就会自己冲去写
* 操作浏览器和操作 APP，不一定适合用同一种模型

关于扩展，他反复提醒：

* 不要从网上乱下
* 尽量走系统提供的、本地加载的、可控的扩展路径
* 需要时就开开发者模式、本地加载、再配 token

这场里还多了一些很容易被忽略的小坑：

* 切模型以后，行为会明显变
* 有的版本会自动 fallback 到别的模型
* 老重启会话、老重置环境，表现也会变
* 某些 profile / 分组设置会影响它能不能控制同一组里的网页或软件

## [[浏览器控制与远程模式]]

浏览器这一段不只是“AI 帮你点按钮”，老师其实讲了两条线。

第一条线是普通浏览器自动化：

* 打开网页
* 在已登录页面里输入需求
* 帮你生成设计稿、内容或代码
* 在网页工具里完成操作

第二条线是更重的远程工作流：

* 在远程环境里启动服务
* 截图
* 创建 PR
* 自动 review
* 让任务挂在仓库流程里继续跑

这场里比之前更明确的几个细节：

* 有状态浏览器和空白浏览器不是一回事
* 有时必须明确说“打开我当前控制的浏览器”
* 某些插件版本需要把窗口放同一组里
* Mac 端支持通常比 Windows 顺
* 本地插件版本和线上版本不一定一致

## [[APP自动化]]

老师重点演示了怎么操作微信，这一段最大的价值不在“做成了什么”，而在“为什么这件事这么不稳定”。

他给出的底层解释是：

* 现在很多 APP 自动化不是官方接口
* 主要还是靠截图、坐标、识别位置再点击
* 所以天然就更慢、更烧 token，也更容易被应用拦

他反复强调的实操经验包括：

* 微信封得很凶
* 飞书、QQ 更容易进入正式工作流
* 权限没开，APP 就根本驱不动
* 只用自然语言不稳定，步骤型指令更稳
* 同一个 case 在不同模型上成功率差很多

## [[VSCode自动化]]

VS Code 这一段，老师其实是在讲“软件自动化如何从浏览器过渡到开发工具”。

他的常见流程大概是：

1. 打开 VS Code
2. 打开指定项目目录
3. 打开 code / codex 一类扩展
4. 等扩展激活
5. 在输入框里粘贴需求
6. 回车
7. 让它自己开始干活

但这场里补了几个容易忘的细节：

* 指令写得越细，成功率越高
* 只写自然语言，模型可能误解
* 编码模型会绕过你原本想走的 UI 路线，直接下场改代码
* 如果你把权限开成完全允许，后面它会更连续地执行
* 如果想更丝滑，最好还是优先靠官方 docs / scopes / 原生能力，而不是纯截图坐标

## [[agent军团]]

这场直播的真正终点不是“会自动开个微信”，而是把前面的东西都收进 agent 军团。

老师现场搭的大致是 5 个角色：

* 会议需求开发官
* 前端情报官
* 舆论 / 币圈观察官
* 抖音二创剧本官
* 健康睡眠心情官

他最想讲的是：

* 多个 agent 可以并行
* 每个 agent 可以有独立上下文
* 你可以在手机上调它们
* 你的目标不是“坐在电脑前省两步”，而是离开电脑以后它们还能继续工作

这场里额外补出的关键细节有：

* agent 如果没落盘，重启后可能就没了
* 有的状态只在内存里
* 有的会写到 workspace 或 `jobs.json`
* 不同模型建 sub-agent 的方式不一样
* 有的模型只给方案，不真正帮你建出来

## [[联网搜索与API Key]]

这场后半程几乎都在排查联网问题，这部分非常值得单独记。

老师现场来回切了这些方案：

* Web search
* 浏览器直接访问网页
* GitHub 公用 API
* 社区 API
* Brave Search 一类搜索 key
* 模型自身的联网能力

这场会真正想说明的是：

* 联网失败不一定是模型不行
* 很多时候是网关、代理、key、权限、配置文件的问题
* 免费方案不是没有，但很容易限流、失效或装不上
* 浏览器直开网页，有时比 web fetch 更现实

老师还明确提到：

* 有些模型自己就有联网能力
* 但如果环境把它拦住，它也做不到
* 它做不到时，更多是在报告“门没开”，而不是胡编结果

## 这场里容易漏掉的补充细节

### 1. 远程环境已经很接近“自动提 PR + 自动 review”

直播一开始其实演示了很关键的一段：

* 启动服务
* 截图
* 生成 PR
* 自动 review

这说明老师不是只把 AI 当“对话工具”，而是在把它接进代码协作链路。

### 2. 版本、分组、登录态是浏览器控制的三大变量

这一场比之前更强调这几点：

* 插件版本不同，效果差很多
* 窗口有时必须放同一组
* 登录态浏览器和空浏览器完全不是同一回事

### 3. 老重启，行为会变

老师反复现场踩到一个坑：

* 一旦重启
* 会话可能切新
* 记忆可能丢
* 模型 fallback 可能变
* 原本能用的 agent / 指令可能又不一样了

### 4. 这场其实已经把“离开电脑工作”的目标说得很明确

老师这场多次重复：

* 坐在电脑前，有些事情你自己做更快
* 真正的价值是你人不在电脑前，它还能继续跑
* 前面所有技能最后都是为这个目标服务

### 5. 云和 AWS 没展开，是因为时间被排障吃掉了

老师中后段明确说：

* 本来今天想讲 AWS 云
* 但这节被联网和工具排障拖住了
* 下周会开始更多接触代码和全栈

## 课程推进和作业提醒

老师在工具演示之外，也一直在把话题拉回课程推进：

* 春节作业要继续补
* 要交录屏 / 视频
* 这套活到毕业时必须熟
* 否则现在找工作会越来越难

他给出的职业判断仍然很一致：

* 纯前端空间在缩
* 全栈、自动化、AI 工作流的重要性在上升

## 可执行版总结

如果把这场直播压缩成真正值得执行的版本，可以记成：

1. 先把主力模型和扩展装法整明白
2. 先跑通一个浏览器自动化场景
3. 再跑通一个 APP 自动化场景
4. 再跑通一个 VS Code 自动化场景
5. 开始给常用 agent 写步骤化提示词
6. 把联网、key、代理、网关这些基础设施配稳
7. 最终把这些能力收束成可并行的 agent 军团
